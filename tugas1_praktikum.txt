1. Apa yang dimaksud dengan dokumen root? 
jawab : Dokumen root adalah direktori utama di mana file-file web disimpan dan diakses oleh server web. Ini adalah lokasi di mana server web mencari file untuk ditampilkan kepada pengguna ketika mereka mengakses situs web. Dalam konteks server Apache, dokumen root biasanya berisi file HTML, CSS, JavaScript, dan file lainnya yang diperlukan untuk menjalankan situs web.

2. Pada Apache web server, apakah dokumen root harus terletak dalam direktori C:\xampp\htdocs? Jelaskan jawaban anda. 
jawab : Tidak, dokumen root tidak harus terletak di direktori C:\xampp\htdocs. Meskipun secara default, XAMPP mengatur dokumen root Apache di C:\xampp\htdocs, Anda dapat mengubah konfigurasi Apache untuk menunjuk ke direktori lain sesuai kebutuhan. Ini dapat dilakukan dengan mengedit file konfigurasi Apache (httpd.conf) dan mengubah direktif DocumentRoot untuk menunjuk ke lokasi yang diinginkan.

3. Jelaskan apa yang dimaksud dengan perintah git: init, clone, remote, add, commit, push, dan checkout.
jawab :
- init: Perintah ini digunakan untuk menginisialisasi repository Git baru di direktori saat ini. Ini membuat subdirektori .git yang berisi semua file yang diperlukan untuk repository.
- clone: Perintah ini digunakan untuk membuat salinan lokal dari repository yang ada di remote. Ini mengunduh semua file dan riwayat commit dari repository remote ke komputer lokal.
- remote: Perintah ini digunakan untuk mengelola koneksi ke repository remote. Anda dapat menambahkan, menghapus, atau mengubah URL dari remote repository.
- add: Perintah ini digunakan untuk menambahkan perubahan (file baru, file yang dimodifikasi) ke staging area, sehingga siap untuk di-commit.
- commit: Perintah ini digunakan untuk menyimpan perubahan yang telah di-stage ke dalam repository lokal dengan pesan yang menjelaskan perubahan tersebut.
- push: Perintah ini digunakan untuk meng-upload commit yang ada di branch lokal ke branch yang sesuai di remote repository.
- checkout: Perintah ini digunakan untuk beralih antara branch yang berbeda atau untuk mengembalikan file ke keadaan tertentu dalam riwayat commit.

4. Jelaskan perbedaan antara GitHub dan GitLab. 
jawab : GitHub dan GitLab adalah platform untuk hosting repository Git, tetapi ada beberapa perbedaan utama:
- GitHub lebih fokus pada kolaborasi dan berbagi kode, dengan fitur seperti pull request dan GitHub Actions untuk CI/CD. GitLab menawarkan fitur yang lebih lengkap untuk DevOps, termasuk CI/CD terintegrasi, manajemen proyek, dan pelacakan masalah.
- GitHub adalah layanan berbasis cloud, sedangkan GitLab dapat di-host secara lokal (self-hosted) atau digunakan sebagai layanan cloud.

5. Apa bedanya variable user dengan variable system? 
jawab : Variable user adalah variabel yang didefinisikan oleh pengguna dan dapat diubah sesuai kebutuhan pengguna. Contohnya adalah variabel yang menyimpan preferensi pengguna dalam aplikasi. Sementara itu, variable system adalah variabel yang didefinisikan oleh sistem operasi dan biasanya tidak dapat diubah oleh pengguna. Contohnya adalah variabel yang menyimpan informasi tentang lingkungan sistem, seperti PATH.

6. Apa persamaan variable system dengan variable constant? 
jawab : Persamaan antara variable system dan variable constant adalah keduanya tidak dapat diubah setelah didefinisikan. Variable system memiliki nilai yang ditentukan oleh sistem operasi dan tidak dapat diubah oleh pengguna, sedangkan variable constant adalah nilai yang ditetapkan dalam program dan tidak dapat diubah selama eksekusi program. Keduanya berfungsi untuk memberikan informasi yang stabil dan konsisten.
